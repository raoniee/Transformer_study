- 왼쪽은 트랜스포머의 인코더 부분, 입력값이 들어오는 부분
- 어텐션 층과 순방향(feedforward)층으로 이루어짐

- 오른쪽은 트랜스포머의 디코더 부분, 타깃 출력값을 입력 받음
- 두 어텐션 층과 하나의 순방향층으로 이루어짐
- RNN, LSTM, CNN 등은 전혀 사용하지 않음 => 재귀적 구조가 없다.

- 왜 재귀적 구조대신 어텐션을 썻냐?
- 재귀적 구조는 단어 간 거리가 멀어질수록 더 많은 파라미터가 필요하다.
- 하지만, 어텐션은 "단어-투-단어(word to word)" 실제로는 "token-to-token"로 재귀적구조의 단점을 커버할 수 있다.

- 그래서 트랜스포머는 각 어텐션 층마다 여덟 번의 어텐션 메커니즘을 병렬로 처리하고,
- 이 구조를 "멀티-헤드 어텐션"이라고 불린다.
- 기능에는
1. 시퀀스에 대한 더 심층적인 분석
2. 재귀적 구조를 없애 계산량 감소
3. 병렬화로 인한 학습 시간 단축
4. 동일한 입력 시퀀스를 다른 관점으로 학습하는 각각의 어텐션 메커니즘

# 2.1.1 인코더 스택
- 인코더 층은 총 N=6개이고 모두 동일한 구조
- 각각의 층에 "멀리-헤드 어텐션 메커니즘", "완전 연결 위치별 순방향 네트워크"인 두 서브 층을 가지고 있다.

- 잔차 연결(residual connection)이 각 서브 층을 둘러싸고 있다.
- 잔차 연결이 하는 역할은?
- 서브 층의 입력 x를 층 정규화(layer nomalization)함수에 전달
- 위치 인코딩(positional encoding)과 같은 중요한 정보가 손실되지 않도록 보장
- 그래서 각 층의 정규화된 츨력은: LayderNomalization(x+Sublayer(x))

- 인코더의 N=6개 층이 모두 완전한 동일한 구조일지라도, 각 층은 서로 다른 내용을 담는다.
- e.g: 임베딩 서브층은 스택의 가장 아래에만 위치 -> 인코딩된 입력이 모든 층에 걸쳐 안정적으로 유지

- 멀티-헤드 어텐션 메커니즘 또한 여섯 개의 층에 동일하게 적용되지만 각자 다른 역할을 수행한다.
- 각 층은 이전 층의 출력을 토대로 시퀀스 내 토큰들의 관계를 파악할 다양한 방법들을 학습한다.
- 멀티-헤드 어텐션 메커니즘은 단어 사이의 다양한 연관성을 찾는다.

- 임베딩 층과 잔차 연결을 포함해서 모든 서브 층의 출력 차원을 일정하게 했다.
- 이 차원(d model)은 목적에 따라 다른 값으로 설정할 수 있음
- 오리지널 트랜스포머 모델은 d model=512로 설정

- 출력 차원 d model을 일정하게 유지하면 -> 연산량과 리소스의 사용량은 줄고 -> 모델에 흐르는 정보를 쉽게 추적

## 2.1.1.1 입력 임베딩
- 입력 임베딩 서브층은 오리지널 트랜스포머 모델의 학습된 임베딩을 사용하여
- 입력 토큰을 d model=512차원의 벡터로 변환

- 임베딩 서브층은 일반적인 트랜스덕션 모델과 동일하게 동작 -> 토크나이저가 문장을 토큰으로 분리
- 토크나이저의 예: BPE, 워드 피스, 센텐스 피스 (오리지널 트랜스포머는 BPE를 씀)
- "the Transformer is ~" 토크나이저를 하면 -> ['the', 'transformer']  
=> 대문자를 소문자로 분리하고 + 문장을 하위 부분들로 잘라냄  
++ 그리고 보통 임베딩과정에서 사용될 정수 표현까지 제공

- 현재 토큰화된 텍스트로는 활용할 정보가 충분치 않다.
- 토큰화된 텍스트를 임베딩해야 한다.
- 그래서 트랜스포머는 학습된 임베딩 서브 층을 포함한다.

- 임베팅 서브층을 설명하기 위해, 구글의 스킵 그램 아키텍처를 설명한다.
- 스킵 그랩: 주어진 단어에 기초하여 주변(context)단어를 예측하도록 학습하는 모델
- 예를 들어, 스탭 크기가 2인 window의 중심에 단어 word(i)가 있다면,
- word(i-2), word(i-1), word(i+1), word(i+2)를 학습하고
- 윈도우를 한 칸씩 움직이며 과정을 반복한다.
- 스킵 그램 모델은 입력층, 가중치, 은닉 층, 토큰화된 입력단어에 대한 임베딩 출력 층으로 구성

- 코사인 유사도(cosine similarity)
- 역할: 두 단어의 임베딩 결과를 검증 할 수 있다.
- 방식: 백터를 표현하기 위해 유클리드 노름을 사용한다.

- 예시로 'black'과 'brown'에 대한 두 d model=512차원 임베딩 벡터의 코사인 유사도를 구현해보자
- => consine_similarity(black,brown)=[[0.9998901]]
- 결과: 스킨 그램이 서로 유사한 두 백터를 생성했음을 알 수 있고, 색상의 하위 집합임을 감지함

### 위치 인코딩
- 입력 임베딩에 위치 인코딩 값을 더하여 시퀀스 내 위치를 표현한다.
- 왜? 위치 벡터를 별개로 학습하면 트랜스포머의 학습 속도가 매우 느려질 수 있고,
- 어텐션의 서브 층이 너무 복잡해질 위험이 있기때문

- 위치 임베딩을 구현하는 방법에는 여러가지가 있지만, 트랜스포머 설계자의 방식으로 단위구와 사인(sin) 및 코사인(cos)으로 구현한다.
- "The black cat sat on the couch and the brown dog slept on the rug."
- 'black'은 pos=2, 'brown'은 pos=10에 위치한다.
- 정보가 더해져야 할 벡터의 크기는 d model=512차원이므로, 512개의 숫자를 사용해 단어 임베딩 벡터에 위치정보를 줘야한다.
- 단어 임베딩의 맨 앞 차원부터 시작해서 i=0부터 i=511까지 순서대로 적용한다.
- 짝수 번째에는 사인함수를, 홀수 번째에는 코사인함수를 적용한다.

- 위치 백터 코사인 유사도: cosine_similarity(PE(2), PE(10)) = [[0.860013]] 
- 단어 코사인 유사도: cosine_similarity(black, brown) = [[0.9998901]]
- 위치 백터의 유사도가 단어 임베딩의 유사도보다 낮은 값을 보여준다.
- 이를 통해, 위치 백터로 두 단어를 서로 떨어뜨릴 수 있는 것이다.
- 다만, 단어 임베딩은 학습에 사용되었던 말뭉치에 따라 달라질 수 도 있는 점을 유의하자.
